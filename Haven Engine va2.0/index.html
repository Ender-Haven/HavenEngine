Planned Features / Q&A

Planned features:

  • Main editor | The centerpeice of the project.
    • Data Structure | The core structure that handles data.
      • Object | A generic object datatype with x, y, w, h, and some flags.
        • Colliding | An object with collision.
        • Static | An object which does not move.
        • Movable | An object that is capable of moving.
        • Overlap | An object that detects other objects overlapping it.
        • Transparent | For lighting purposes
        • Transparent Color | For lighting purposes
        • Entity | An object that simulates physics
          • Player | An entity controled by a player.
          • AI | An otherwise player controlable entity controlled by an AI.
          • Pushable Object | An object that can be moved by other objects.
          • Projectile | An object launched or dropped, or otherwise something small moving at a high speed.
        • Interactable | An object that the player can click on or press a button near.
      • Engine Manager | NOT THE ENGINE! The core bit that helps with compatability, including backwards compatabilty and forward compatability.
        • Engine Loader | Loads a non-integrated, perhaps modded engine or plugin.
        • Global Consistancy Patcher | Fixes old data by updating it, or makes newer objects convert for older engines.
      • Level Manager | Handles the easy grid-based level editor.
        • Object Grid | The grid system for the level.
        • Level compressor | A system for reducing the ammount of colliding objects without losing quality.
    • Integrated Engine | The actual renderer and physics engine for the game.
      • Renderer | The system that displays the game on screen.
        • Generic Layer | Unshaded images or shapes.
        • Shader | Shading overlay and shaded images or shapes.
          • Pre-built lighting | Simple light casting that stops upon hitting a solid object, usually cast from multiple points on the light source.
          • Dynamic lighting | Simple light casting that stops upon hitting a solid object
          • Normals | A texture overlay to simulate 3d items.
          • 2d Ray Traced | Rays are cast and bounce off of surfaces, spreading the color and reflecting.
            • Dynamic Ray Tracing | Few rays cast from one point
            • Built Ray Tracing | Multiple rays cast from many points
      • Physics Engine
        • Engine1 Generic 2d physics | Simple 2d physics without rotation.
        • Engine2 Advanced 2d physics | Advanced 2d physics including rolling.
        • Engine3 Complex 2d physics | Highly complex simulations including fluid simulation and buoyancy
    • Console | Display text, hopefully with colors. :)
  • Logic Editor | An editor for making new gameplay elements easier. FUTURE
  • Documentation | A basic text guide. FUTURE
  • HAVEN Assistant | A live programming assistant that can walk you through creating a game. FUTURE

<!> SOUND WARNING <!>

This application uses sound effects and voiceover as part of the editor. It's not required in order to use the engine, but some(non-vital) features may be unavalible or seemingly broken.

I'll work on adding sound-less versions of all the sound features (using some sort of transcript feature, most likely.)

Use sound Defaults to 20% volume. We don't want to blast your ears. Disable sound Sound will still be present when you run your game.
You can enable editor sounds later in the editor settings.
